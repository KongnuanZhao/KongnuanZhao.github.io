      Post on Mar 17 2015 By 赵空暖

      郁闷一下午提交一直是Everything up-to-date,要不就是

      先记录一下今天遇到的问题：
      使用hexo写博客源文件都是在本地的，如果换了电脑需要写博客时就会比较麻烦。
      我们需要在电脑上新建一个文件夹如:111,然后进入这个文件夹克隆创建仓库。

      设置远程仓库地址，

      origin可以改为别的名字，相应的改变在.git文件下的Config配置文件

      如果要是执行git remote add gitcafe git@gitcafe.com:KongnuanZhao/KongnuanZhao.git,该配置文件就会添加：

      查看你当前项目远程连接的是哪个仓库地址:git remote -v

      之后，当写博客后，只需要git add .，将所有文件提交到缓存区。
      git commit -m "add all files" ，将这些文件提交到本地仓库。
      git push origin master，将本地仓库的改动推送到github仓库。
      当远程仓库有更新时，使用git pull或者git fetch就可以同步代码到本地了。
      从远程库抓取信息,然后将远端分支自动合并到本地仓库中当前分支。这么用，既快且好。默认情况下git clone命令本质上就是自动创建了本地的master分支用于跟踪远程仓库中的 master 分支。所以一般我们运行git pull，目的都是要从原始克隆的远端仓库中抓取数据后，合并到工作目录中的当前分支。

      再就是今天莫名其妙的发现不能同时提交代码到github和gitcafe上了，没办法，只能记录下下面这条命令喽。
      先查看远程仓库地址

      到\Hexo\.deploy目录底下，执行（现在也不需要切换分支也能提交啦）
      git checkout gitcafe-pages && git push gitcafe gitcafe-pages && git checkout master && cd .. && hexo d
